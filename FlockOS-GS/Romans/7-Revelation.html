The HTML Applications

The Good Shepherd (Admin Console)

This is the main application church staff use daily. It includes:

  • Role-gated sidebar navigation — modules appear based on the user's role level
  • Dashboard with quick-access cards — common tasks presented as clickable cards
  • Notification system — bell icon with real-time notifications, click-to-navigate
  • Module toggle system — administrators can enable/disable modules per church needs
  • Favorites bar — users can pin their most-used modules
  • Theme selector — 8 visual themes to match church branding

The Public App (index.html in root)

The congregation-facing app lives in the project root. It provides devotionals, reading plans, Bible quiz, prayer requests, event registration, giving, and community features — all without requiring admin access.

Deployment Process Overview

  1. Create 4 Google Sheets (Matthew, Mark, Luke, John)
  2. Create 4 Apps Script projects (one per Sheet)
  3. Copy .gs files into the correct projects
  4. Run setup functions to create all 200 tabs
  5. Deploy each as a Web App
  6. Configure API endpoints in the frontend HTML files
  7. Set up the first admin account
  8. Deploy the HTML applications (host or serve via Apps Script)

Full details are in 3_Deployment.txt.

The Wiring Audit

The 8_Wiring.txt file in this folder documents every frontend-to-backend connection in the system. It maps which parameters the frontend sends, which parameters the backend expects, and flags any mismatches. This is the single source of truth for maintaining the system's integrity during development and expansion.
