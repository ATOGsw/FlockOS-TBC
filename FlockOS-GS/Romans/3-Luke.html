Purpose

The Luke API is FlockOS's analytics and expansion engine. It houses 53 tabs dedicated to statistical tracking, trend analysis, engagement metrics, and growth dashboards. It also serves as the expansion framework — pre-built slots for future features that can be activated without restructuring the system.

Files in This Folder

FilePurpose
Luke.gsEntry point — doGet() handles health checks and routes API requests to the statistics data tabs.
Extra.gsSetup script — setupExtraApi() creates all 53 tabs with headers and analytics-ready structures.

Database: 53 Tabs

The Luke Sheet is organized into analytics domains and expansion reserves:

Church Growth Metrics

  • WeeklyAttendance — Aggregate attendance numbers by week and service type
  • MembershipGrowth — New members, transfers, departures over time
  • VisitorTracking — First-time guests, return rates, conversion to membership
  • BaptismRecords — Baptism counts and trends

Engagement Analytics

  • GroupParticipation — Small group enrollment and attendance trends
  • VolunteerHours — Service hours logged across ministries
  • GivingTrends — Aggregate financial trends (no individual amounts — those stay in John)
  • EventAttendance — Special event participation rates

Ministry Dashboards

  • CareMetrics — Pastoral care case volumes, resolution times, follow-up rates
  • PrayerMetrics — Prayer request volumes and response times
  • MissionsMetrics — Outreach activity aggregates
  • CommunicationsMetrics — Message delivery and engagement rates

Expansion Slots

Luke includes pre-provisioned expansion tabs — empty structures ready for future features. When you need a new data domain, activate the slot rather than creating tabs from scratch. This keeps the system organized and predictable.

Implementation for Your Church

  1. Create a Google Sheet — this will be your Luke database
  2. Open Apps Script — Extensions → Apps Script
  3. Copy Luke.gs and Extra.gs into the script editor
  4. Run setupExtraApi() — creates all 53 tabs
  5. Connect to your CRM — the Statistics module in the frontend reads from both Luke and John APIs to generate dashboards
  6. Deploy as Web App — set access to match your John API (authenticated)

Use Cases

  • Board meetings — generate at-a-glance growth dashboards for church leadership
  • Ministry evaluation — track which programs are thriving and which need attention
  • Budget planning — use giving trends to forecast and allocate resources
  • New feature development — use expansion slots to prototype new data domains quickly

Why a Separate Database?

Statistics live in their own Sheet (separate from the main CRM) for three reasons:

  • Performance — aggregate queries don't compete with transactional CRM operations
  • Access control — analytics can have different permission levels than membership data
  • Scalability — the expansion slots allow the system to grow without touching the core CRM
