Purpose

The Matthew API serves public-facing, read-only content to the congregation. It powers the member app's devotional system, Bible reference tools, reading plans, theology articles, and quiz engine. No authentication is required — this is the church's open resource library.

How the API Works

The Matthew API is a simple tab reader. When the frontend calls TheVine.app.books(), it sends a GET request with the tab name. Matthew.gs reads the corresponding sheet tab and returns the rows as JSON. There is no write access — Matthew is strictly read-only.

API Pattern

GET ?tab=Books → Returns all rows from the Books tab as an array of objects
GET ?tab=QuizQuestions → Returns all quiz questions
GET ?tab=Devotionals&date=2026-03-21 → Returns today's devotional

Implementation for Your Church

  1. Create a Google Sheet — this will be your Matthew database
  2. Open Apps Script — Extensions → Apps Script in the Sheet
  3. Copy Matthew.gs and Truth.gs into the script editor
  4. Run setupAppApi() — this creates all 12 tabs with proper headers
  5. Populate content — add your devotionals, theology articles, and quiz questions
  6. Deploy as Web App — set access to "Anyone" (no authentication needed)
  7. Note the URL — this is your APP API endpoint for frontend configuration

Content Tips

  • The Books tab comes pre-seeded with all 66 books — edit summaries to match your church's teaching style
  • Devotionals can be written by multiple authors — use the Author column
  • Quiz questions support multiple choice (4 options) with scriptural references
  • Reading plans are modular — create as many plans as you want, each with its own day-by-day breakdown
