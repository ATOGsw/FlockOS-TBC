What Is FlockOS?

FlockOS is an open-source Church Relationship Management (CRM) system designed for churches of any size. It runs entirely on Google's free infrastructure — Google Sheets as the database, Google Apps Script as the backend, and static HTML/JS as the frontend. There are no recurring hosting costs, no complex server setups, and no proprietary data formats.

FlockOS manages your entire church operations: membership directory, attendance tracking, pastoral care, communications, giving records, event management, worship planning, missions, small groups, and much more — across 200 spreadsheet tabs organized into four purpose-built databases.

📚

Public-facing content: devotionals, Bible books encyclopedia, reading plans, lexicon, quiz questions, theology articles, and apologetics.

🌎

The heart of operations: members, authentication, pastoral care, services, songs, communications, giving, events, and 79 tabs of relational data.

How It Works

Architecture overview: FlockOS uses four Google Sheets as databases (one per API), each exposed through a Google Apps Script Web App. The frontend is a single-page application (SPA) built in vanilla JavaScript — no frameworks, no build tools, no dependencies.

The Four APIs

Each API is a standalone Google Apps Script project attached to its own Google Sheet:

  • FLOCK (John) — 79 tabs. Members, auth, pastoral care, services, songs, communications, giving, events, groups, volunteers, and more. This is the primary CRM backend.
  • APP (Matthew) — 12 tabs. Public content: devotionals, Bible encyclopedia, reading plans, quiz, theology. Read-only, no authentication required.
  • MISSIONS (Mark) — 56 tabs. Persecution data, country dossiers, mission teams, field reports. Specialized global outreach module.
  • EXTRA (Luke) — 53 tabs. Analytics, statistics, and expansion slots for future features.

Frontend Layer

Two main HTML applications serve different audiences:

  • The Good Shepherd (admin) — Full CRM with 48+ modules, role-gated navigation, notification system, and dashboard cards. For church staff and volunteers.
  • Public Site (index.html) — Member-facing app with devotionals, reading plans, prayer requests, learning hub, ministry hub, and community features. Auth-scoped navigation reveals modules as users log in.

Getting Started

  1. Create four Google Sheets — one for each API (Matthew, Mark, Luke, John). These will be your databases.
  2. Create four Apps Script projects — attach each to its corresponding Sheet via Extensions → Apps Script.
  3. Copy the .gs files — place them in the correct Apps Script project according to the folder structure (1-Matthew files go in the Matthew project, etc.).
  4. Run the setup functions — each project has a setup function that creates all the necessary tabs and headers automatically.
  5. Deploy as Web Apps — publish each Apps Script project. Set access to "Anyone" for public APIs (Matthew) or "Anyone with Google account" for authenticated APIs (John).
  6. Configure the frontend — update the API endpoints in the frontend configuration to point to your deployed Web App URLs.
  7. Create your first admin account — run the initial user setup to create a pastor-level admin account.
  8. Invite your team — add staff and volunteers with appropriate role levels.

Security

FlockOS implements enterprise-grade security on free infrastructure:

  • SHA-256 with salt & pepper — passwords are never stored in plain text
  • 6-tier RBAC — readonly, volunteer, care, leader, pastor, admin
  • Session tokens — time-limited authentication tokens with automatic expiry
  • Confidential case gating — sensitive pastoral care cases restricted to pastor+ roles
  • Audit logging — every data modification is logged with user, timestamp, and action
